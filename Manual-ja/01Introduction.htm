1. はじめに

SWCLOS は Lisp で知識処理をしたい リスププログラマーのためのソフトウェアツールである． その名前は Semantic Web Common Lisp Object System の頭文字からとられているが，その名の示す通り SWCLOS はセマンティックウェブのために，Common Lisp Object System (CLOS)上に構築されている． SWCLOS においては，RDF, RDFS, OWL, そしてユーザ定義のオントロジーにおける語彙のすべての 用語はCLOSのオブジェクトを表示し，その意味論はCLOSではなく RDF(S) と OWL に基づいているが， プログラミングは依然としてオブジェクト指向のパラダイムに従っている．

SWCLOS はセマンティックウェブ技術とオブジェクト指向プログラミング技術を融合させたものである． その他のセマンティックウェブ・システムに比べて，SWCLOS にはユニークな特徴がある． SWCLOS の利点も欠点も，動的オブジェクト指向言語である CLOS と Lisp 言語の性質に帰することが できると言えよう． もしリスププログラマーがセマンティック技術のためのアプリケーションを開発 する必要があるとすれば， SWCLOS はその解となり得る． 更に，もしそのプログラマーが CLOS プログラミング に長けているのなら， SWCLOS は最適の解となる． SWCLOS は CLOS のメタプログラミング機能を用いて， OWL Full レベルでのオントロジー・メタモデリング機能を提供する． リスププログラマーはいつもの プログラミングと同様なやりかたで， SWCLOS 上でセマンティックウェブのための CLOS プログラミングを 行うことができる． 必要なことは，標準の CLOS 意味論と RDF(S)/OWL 意味論の違いを学ぶことだけである．

この文書は SWCLOS プログラマーのためのユーザマニュアルである． SWCLOS のプログラムは RDF/XML フォーマット， N-トリプルフォーマット， ttl フォーマット， そして SWCLOS 用のリスプの S 式のオントロジーファイルを読み込むことができ， リスプで SWCLOS オブジェクトを取り扱って，その処理結果を上記の様々なフォーマットに書きだすことができる． それゆえ，そのようなセマンティックウェブの各種フォーマットを介して他とコミュニケートすることができる．

もし CLOS プログラマーがセマンティック技術とは何かを知りたいのなら，SWCLOS はそれを学んで セマンティックウェブ・プログラミングを開始する絶好の機会を提供する．
