2.  SWCLOS の基本

2.1  SWCLOS を入手する

SWCLOS は当初は開発者である小出誠二によって，株式会社ギャラクシー・エクスプレスにおいて開発された オープン・ソース・プログラムであり，引き続き彼によって更新が継続されている． 最新の版は国立情報学研究所 (NII) の彼のホームページ http://www-kasm.nii.ac.jp/~koide/SWCLOS2.htm から入手可能である．

SWCLOS の現在の版は Franz 社の Allegro Common Lisp (ACL) 8.1, 8.2, 9.0 上で開発されている． だれかによって他のリスプ処理系に移植されることを望むものではあるが，その作業は困難を極めるであろう． なぜならば，一部の実装は ACL 特有の部分に深く関係しているからである．

2.2  権利関係

株式会社ギャラクシー・エクスプレスがかつてすべてのコピーライトを所有していて， パッケージ名 gx と gx-user を変更すること以外は， AS IS で SWCLOS の利用を制限なしに許可していた． 'GX’はギャラクシー・エクスプレスが開発予定であったロケットの名前である． 現在の状況については下記の注記と配布の README ファイルを参照されたい．

株式会社ギャラクシー・エクスプレスは 2010 年に株式会社 IHI に吸収され， 開発者の小出誠二も IHI に移動した．したがって，上記の条項は IHI に引き継がれ， 現在もギャラクシーに代えて IHI として効力を持っていると考えられる．その後， 小出は IHI を退職し，現在は国立情報学研究所の特任研究員である． SWCLOS は日本の IT プログラム 「IT を用いた大規模システムのための運転支援システムの構築」 の成果の一つである．

2.3  SWCLOS を始めるには

SWCLOS を利用するには， ACL modern リスプに SWCLOS をコンパイル・ロードののち，リスプリスナー・ウィンドウにおいて gx-user パッケージに入るのがよい． SWCLOS では大文字小文字を区別し，リスプのリーダは case-sensitive-lower モードでなければならないことに注意されたい．この仕様はANSI-標準ではないが，間もなくそうなることを期待している．

cg-user(1): (in-package gx-user)
#<The gx-user package>
gx-user(2):

Franz 社は Allegro Common Lisp における case-sensitive 機能について，よいチュートリアルを提供している． http://www.franz.com/support/tutorials/casemode-tutorial.htm を参照されたい．

2.4  知識の単調性について

セマンティックウェブでは知識は常に増加し，解釈はいつでもより精密により豊かになることが前提とされている． SWCLOS ではオントロジーにおける前方参照の解決のために，この単調性の原則を利用している． 現在の版では，オントロジーにおけるエンティティが削除されることはなく，undo 機能は提供されていない （もし矛盾する知識が入力されると， SWCLOS はすぐに警告を発してブレークするので，その場ですぐに問題を修正することができる）． Undo のような機能は次期バージョンにおいて提供されるであろう．

SWCLOS の至る所で， その機能はセマンティックウェブの原則にしたがっているが， ただし，上記単調性の原則によりスロット値に新しい値を追加するには，put-value 関数を用いなければならない． 通常のスロット値設定機能 (setf (slot-value ...) ...) はこの版では元の CLOS の意味論を保持している．

2.5  開世界仮説と三値論理

セマンティックウェブでは World Wide Web は動的であり，移ろいやすく， WWW のすべてを知ることは不可能であるという意味で，オープンとされている． 論理の観点から言えば，閉世界仮説は推論の基本的前提である．それゆえ， 論理の考えかたに慣れた人はこのことを心に留めるべきである．

記述論理 (Description Logic, DL) の主な機能は, i) オブジェクトをクラスに分類すること, ii) あるクラスが他のクラスに包摂されるかどうかを決定すること, である． SWCLOS ではこの働きは, i) gx:typep により, ii) RDF ユニバースにおける gx:subtypep, OWL ユニバースにおける gx:subsumed-p により，実行されるが，これらの関数は返値として二つの値を返す． 最初の値はクエリに対する yes/no を表現し，二番目の値は確実さを表現する． それゆえ, <t,t> は論理値の真を表し，<nil,t> は偽を表すと見なすことができる． そして, <nil,nil> は未定または未知を意味する．<t,nil> は決して起こらない． この三値論理は元々上記の値を返す cl:subtypep についての ANSI Common Lisp の仕様を拡張したものである．その詳細は ANSI Common Lisp 仕様書 http://www.lispworks.com/documentation/HyperSpec/Body/f_subtpp.htm#subtypep を参照されたい． しかしながら，リスププログラマーはこの三値論理の使い方に惑う必要はない．もし返値の最初の値だけを使うのなら， プログラムは未定を偽と同等に扱うことになる． SWCLOS において三値論理をどう計算しているのか興味があれば， お手元のソースコードを参考にされたい．
